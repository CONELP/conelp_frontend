# CONELP Frontend

공사현장 문서 자동 생성을 위한 프론트엔드 프로젝트입니다.
사용자가 현장 자료(주로 이미지)를 업로드하면, 백엔드에서 OCR 및 문서 생성 처리를 수행하고, 프론트엔드는 업로드부터 결과 확인/다운로드까지의 사용자 경험을 제공합니다.

## 프로젝트 목표

- 복잡한 현장 문서 작업을 간단한 업로드 흐름으로 단순화
- 문서 종류별 입력 차이를 감추고 일관된 생성 경험 제공
- 생성 완료된 문서를 한 곳에서 조회, 다운로드, 관리할 수 있는 화면 제공

## 문서 종류

- 공사일보
- 자재반입 검수요청서 (철근)
- 콘크리트 반입 + CSI 입력
- 콘크리트 압축강도 (현재 미지원, 추후 추가 예정) + CSI 입력
- 검측 요청서

## 핵심 사용자 흐름

1. 문서 종류 선택
2. 자료 업로드
3. 생성 진행 상태 확인
4. 완료 결과 확인 및 다운로드
5. 생성된 문서 목록 관리

## 이번 단계에서 다루는 범위

- 프론트엔드 UI/UX 구조 정의
- 업로드 시퀀스 설계
- 문서 관리 화면 설계
- 상태별 사용자 메시지와 기본 인터랙션 정리

백엔드 문서 처리 기능은 이미 존재한다고 가정하며, 현재 저장소에서는 프론트엔드 기획과 화면 설계를 우선합니다.

## 향후 개발 메모

- 문서별 업로드 요구사항 차이를 UI에서 명확히 안내
- 모바일 촬영 이미지 업로드를 고려한 UX 필요
- 실패/재시도/부분 누락 데이터 처리 경험 정의 필요
- 엑셀 결과물 외 추가 결과 형식(PDF, 미리보기) 지원 여부 검토 가능
